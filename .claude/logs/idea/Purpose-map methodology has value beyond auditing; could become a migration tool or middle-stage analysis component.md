# Purpose-map methodology has value beyond auditing; could become a migration tool or middle-stage analysis component

## Purpose

The analytical framework — failure-mode framing, need/rationale split, reactive refinement, override-vs-description check, router-vs-destination check — could serve broader uses beyond component justification.

Possible forms:
- **Migration tool** — apply failure-mode framing to validate each piece when moving/restructuring content
- **Middle-stage analysis** — any process evaluating "what does this actually contribute?" can use the framework
- **Zoomable analysis** — works at file → section → bullet granularity

The current purpose-map tool is one SQLite-backed implementation. The methodology behind it is the more reusable artifact. Flag for future exploration after v2 pass stabilizes.
