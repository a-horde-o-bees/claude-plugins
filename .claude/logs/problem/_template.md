# Problem

Observed defects or issues noticed mid-work that need investigation later. Concrete, unexpected, and wrong.

## What Qualifies

A defect the current context noticed but cannot immediately address — wrong output, broken invariant, inconsistent state, performance regression, incorrect behavior.

## What Does Not Qualify

- Rule violations or process gaps → friction
- Ideas for new capabilities → idea
- Settled choices with rationale → decision

## Entry Structure

After the title and purpose statement, include enough detail to investigate: reproduction steps, constraints observed, suspected root cause.

Capture fast. Detail only when substantial context would be lost in a one-line purpose statement.

## Lifecycle

Problems are a queue. Delete entries when resolved or moved to an external tracker.
