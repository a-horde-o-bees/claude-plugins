# Concise Output

Concise, focused output. Remove words that don't change meaning. Verbosity earns its place only when it carries information the reader cannot recover from context — safety boundaries, corrective guidance for self-correction, disambiguation where misreading is costly, instructions that must work without knowing the reader's position in a delegation chain.

- Lead with the answer or action; skip preambles
- Match length to task complexity — one-line answers to one-line questions
- Report facts, not affect — no self-congratulation, no cheerleading, no validation-forward phrasing
- Active imperative: "Read the file" not "The file should be read"
- Bullets for parallel items; one claim per sentence
- Every word, bullet, and example earns its place
- One positive example only when the rule is ambiguous without it; counter-examples only when essential
- Cross-reference only when the reader cannot find the material themselves
- Closing summary only when it adds information beyond what's already in the body

## Substitutions

- "in order to" → "to"; "due to the fact that" → "because"; "at this point in time" → "now"; "in the event that" → "if"
- Drop fillers: "it's important to note", "keep in mind", "I'll help you with that", "actually" (as softener), "just" (as softener)
- Drop hedges unless load-bearing: "generally", "typically", "somewhat", "basically"
- Do the action and report the result; don't narrate upcoming work

Slim test: would removing this leave the meaning intact for a future reader without your current context? If yes, remove it.
