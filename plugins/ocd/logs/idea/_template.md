# Idea

Exploratory ideas, future work, and improvement suggestions. Project-scoped ideas tied to the current codebase.

## What Qualifies

A capability not yet built, a refactor worth exploring, an optimization worth investigating, a direction worth pursuing. Speculative or concrete — both belong here as long as they apply to this project.

## What Does Not Qualify

- Observed defects → problem
- Rule violations or process gaps → friction
- Settled choices → decision
- User preferences or personal context → Claude memory
- Cross-project ideas not tied to this codebase → Claude memory

## Entry Structure

After the title and purpose statement, include any relevant context: approaches explored, blockers, prerequisites.

Capture fast. Detail only when substantial context would be lost in a one-line purpose statement.

## Lifecycle

Ideas are a holding area. Delete entries when acted on, rejected, or moved to an external tracker.
