# Research

External resources organized by domain. Cross-plugin in scope — research that informs any part of the project.

## Format

One file per domain (`testing.md`, `process-flow-notation.md`). Each file contains a title, one-line purpose, and a link list.

Each link entry: URL followed by scope description — what domain the resource covers and what value it provides.

## Description Guidance

Follow the navigator description guidelines model (`plugins/ocd/skills/navigator/references/description-guidelines.md`). Core question each description answers: **should I visit this link?** Convey scope and value so agents can decide relevance without fetching the resource.

- Include scope (what domain the resource covers) and role (what kind of value it provides)
- Exclude content summaries, internal details, or full reproduction of what the link contains
- Quality test: if two links have the same description, descriptions are too vague; if description would change when the resource is updated but its domain stays the same, description is too detailed
