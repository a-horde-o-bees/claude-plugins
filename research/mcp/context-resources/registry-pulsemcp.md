# PulseMCP

## Identification
- url: https://www.pulsemcp.com/
- type: registry / awesome list / blog
- author: PulseMCP (community; maintainers on MCP Steering Committee)
- last-updated: active
- authority level: semi-official — operators include a Steering Committee member (Tadas Antanavicius) and they maintain official integration with MCP's GitHub registry and docs

## Scope
Community hub for MCP: server directory, client directory, use-cases repository, guides/tutorials, and a weekly newsletter. Operators are active community members — Tadas Antanavicius (Steering Committee, contributor community, official registry) and Mike Coughlin (design and maintenance). Submission via "Submit use case," "Submit server/client" forms.

## Takeaway summary
PulseMCP is the closest thing to a "news + ecosystem" hub for MCP. The weekly newsletter is the single best way to track what's shipping in the ecosystem without sampling a dozen repos. Their client directory is one of the few places that aggregates the host side (Claude Desktop, Cursor, Windsurf, Continue, etc.) alongside servers. Because a Steering Committee member operates the site, it functions as a de-facto semi-official complement to modelcontextprotocol.io without being part of it.

## Use for
- How do I stay current with the MCP ecosystem without manual monitoring?
- Where can I find MCP clients (hosts), not just servers?
- Where can I submit a use-case or a client for community visibility?
- Who's actively in the steering/governance side of MCP?

## Relationship to other resources
Adjacent to `registry-official.md` (PulseMCP maintains official integration). Supplements the awesome-list ecosystem with editorial content. Peer to Glama, mcp.so, Smithery on the directory side but differentiated by editorial / newsletter output.

## Quality notes
Editorial content may lag the fastest-moving corners of the ecosystem by up to a week (newsletter cadence). Registry data stays in sync with the official registry.
