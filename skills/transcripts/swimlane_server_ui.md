# Swimlane (live) ‚Äî UI elements

Element reference for the live swimlane page `swimlane_server.py` serves at `/`: every DOM element, its CSS hook, and what it means, plus the encodings (color/shape) that decorate it.

The page is a **flat rail of segments** over a raw transcript DB. One scroll container holds each session as a ts-ordered rail of nodes (root / resume / compaction / rewind, from `branch_tree`) interleaved with expanded event **segments**; all segments share ONE global column layout under a single sticky header, and a right-side detail pane shows the clicked dot. Elements below are nested under their containers (mirroring the DOM); encodings are listed under the element they decorate.

- **Top bar** (`#bar`) ‚Äî global controls
    - **Title + db name** ‚Äî `Swimlane ¬∑ <db file>`
    - **from** / **to** (date inputs) ‚Äî filter the session list to trees overlapping the range (query param on `/api/trees`)
    - **session filter** (`#q`) ‚Äî substring filter over session ids (client-side, instant; re-renders the outline)
    - **apply** (button) ‚Äî re-fetch the trees with the current from/to (also refreshes the Stats/Selection tab if open)
    - **count** (`#count`) ‚Äî `<shown> / <total> sessions`
- **Column header** (`#colhdr`, sticky top of the scroll) ‚Äî the ONE global column header, shared by every segment. Shows **every class present corpus-wide at all times** (min/name width, even with nothing expanded); an expanded segment only **grows** the columns it uses (widths track the open segments, never the whole-corpus outlier). No counts. Sticks vertically, scrolls horizontally with the dots.
    - **column header cell** (`.chdr`) ‚Äî one per present class, at the class's global x; the name **word-wraps and sits bottom-aligned** against the divider (only headers wrap ‚Äî segment contents never do). Hover shows the class description.
    - **membership-lane headers** (`ex` / `th`) ‚Äî two fixed `.chdr` cells **left of the class columns** (right of the gutter), labeling the two membership lanes drawn inside each segment (below). Hover describes them. User-facing bucket names: **user prompt** (prompt), **user answer** (user-response), **user interrupt**, **agent thinking** (thinking), **agent message** (text), tool use, tool result, attachment, **system**, context prep, ‚Ä¶ There is no queue column ‚Äî queue ops dissolve into bridge arcs, and queued prompts surface in the user-prompt column as fate-coded circle/triangle markers (see the node legend).
- **Scroll** (`#scroll`) ‚Äî THE one scroll container; rows, dividers, and dots scroll together (vertical) and share one horizontal scroll. Column widths track only the expanded segments (no outlier blowup). Sized to its content **plus a vertical-scrollbar buffer**, so the content is never clipped into a spurious horizontal scroll, and capped to keep the detail frame ‚â•380px (only a sub-1080 window or many wide open segments forces the left pane to scroll horizontally).
    - **Tree** (`#tree`) ‚Äî a **flat rail**: every node of a session (root + segments + rewinds) on ONE rail in **ts order**, closest to how it happened. The initial path is earliest (top); a rewind lands at *its own* ts (when the rewind was done), below the segments it branched from. No node is "main" and none is a dead-end ‚Äî the **mark** says which kind, the counts say how deep.
        - **row** (`.tnode`) ‚Äî one segment, on the rail (`.railcol` carries the mark over a continuous vertical line). Click toggles **its** event segment (inserted directly below, at x=0). Left‚Üíright:
            - **rail mark** (`.railcol`) ‚Äî ‚óè root/session ¬∑ ‚Ü∫ rewind (amber) ¬∑ ‚á≤ resume segment (blue) ¬∑ ‚ñ£ compaction segment (purple)
            - **chip** (`.chip`, fixed width, color = kind) ‚Äî the segment's **structural kind**, mechanically derived (more diagnostic than who-typed-it): `session` (root) ¬∑ `resume` (blue) ¬∑ `compaction` (purple) ¬∑ `rewind` (amber). Echoes the rail mark. **Compaction vs resume is exact, not a guess:** a segment is `compaction` iff it re-roots at a `compact_boundary` or its lead burst carries an `isCompactSummary` (`_cs`) record ‚Äî a plain resume only re-emits `_replay` copies. `rewind` = a same-file re-prompt fork.
            - **start datetime** (`.dt` ‚Äî `YYYY-MM-DD ¬∑ HH:MM`, fixed width)
            - **text** (`.btext`, flex) ‚Äî **every row's first actual typed prompt** (`first_prompt`), root and segment alike (a session/segment often starts on a non-prompt record, so the first prompt is shown, never the bare hash/start label). The chip and datetime are fixed-width, so the datetime and first-prompt text align straight down the rail ‚Äî the same alignment the right-side fields have.
            - **duration** (`.m.cov`, `h m s` ‚Äî leading empty units dropped, a `0m` kept once an `h` shows, seconds always shown), **prompts** (`.m.prm`), **records** (`.m.rec`) ‚Äî right-justified, aligned across rows, per-segment. Duration = Œ£ the segment's time-block coverage; prompts exclude replays + compact summaries. **Segments** split the one thread at re-emission boundaries (compaction / resume); **rewinds** are divergent re-prompts ‚Äî both ride the same flat rail. The segment's re-attach point, compaction count, and superseded-prompt count ride in the payload but aren't shown inline (recoverable for a future segment-detail surface).
        - **segment** (`.segment`) ‚Äî one expanded segment's events in the shared columns; ordinal pitch, **one row per wall-clock second** ‚Äî records sharing a second share a row (their sub-second ms is unreliable, often inverted vs write/parent order within a burst, so it must not drive vertical position). Within a row they pack into sub-columns in write (line) order, so parent‚Üíchild arcs in a burst stay flat. Height = content. Lazy-fetched from `/api/segment/<start_uuid>` (class-relative geometry; the client assigns x). Rebuilt on any column-width change.
            - **membership lanes** (`.lane`, two thin columns just right of the gutter, left of the class columns) ‚Äî the annotation overlay: one filled cell per row, drawn down to the next row so contiguous rows read as a continuous bar. **ex lane** = the record's described **exchange**, an alternating shade so adjacent exchanges separate. **th lane** = its **focus-thread**, colored from a maximally-distinct **spread palette** (sized to the topic count) ‚Äî **one stripe per shown topic**, so a multi-topic thread shows all its colours side by side. Topics are filtered (and their stripes hidden) from the **Stats tab** (no billable/non-billable grouping; every topic is equally filterable). An **empty cell** = an incidental/unthreaded record (it didn't roll up into a described exchange / a thread); a **broken thread bar** = non-contiguous membership (another thread's exchange interleaves). Click a bar (or any dot) ‚Üí the row's exchange description + thread summary in the Selection tab. Threads are per-root and segments per-file, so a cross-file thread shows only its local slice here; the full summary still appears on select.
            - **left gutter** (x `0 ‚Ä¶ LGUT`, =200) ‚Äî elapsed-time bars + prompt/interrupt time labels; the only time signal (ordinal y is not real-time)
                - **gutter bar** (`.gb`) + **label** (`.glbl`) ‚Äî elapsed time since the previous record; log-scaled length, **heat-colored server-side from the actual seconds** (`_heat`, carried in the payload): dark `<1s` ¬∑ blue `<1m` ¬∑ amber `< ceiling` ¬∑ red `‚â• ceiling`. **The amber‚Üíred boundary is the idle-gap ceiling** (default 300s), so tuning the ceiling moves the red coloring with it; labeled from 60s.
            - **arc layer** (`svg.arcs`) ‚Äî record relationships in this segment, drawn with pronounced curvature; **all arcs share one stroke weight** (the queue bridge is distinguished by color, not thickness) and **queue bridges draw behind** the parentUuid arcs (SVG paint order ‚Äî structural arcs win the overlap)
                - gray = normal parentUuid parent‚Üíchild link
                - red = child timestamp precedes its parent (write-order inversion within a burst)
                - **amber** (`qbridge`) = a dissolved **queue bridge** ‚Äî the queue ops are not drawn; one arc connects the two ends of a chain that bridged: `tool_use ‚Üí delivered notification`, or a hollow queued prompt ‚Üí its delivered prompt. Incomplete chains (removed, slash-command, Monitor, cross-boundary) draw nothing. See `ARCHITECTURE.md` ¬ß *Prompt-queue resolution*
            - **node** (`.nd`) ‚Äî one record, in its class column at its ordinal y; **left-click ‚Üí detail pane**; right-click copies its uuid
                - ‚óè circle = timestamped record; fill = class color
                - **Queued prompts render by FATE in the prompt column, prompt-column color, linked by an amber `qbridge` arc** (the arc shows when it was sent ‚Üí when it was realized):
                    - **‚óã ‚Üí ‚óè** (`.hollow` ring ‚Üí normal record dot) = a queued prompt **emitted as its own prompt event**: hollow circle at the `enqueue`, solid circle at the delivered prompt.
                    - **‚ñ≥ ‚Üí ‚ñ≤** (`.tri`) = a **consumed interjection** the running turn **answered inline** (`enqueue`‚Üí`remove`, never delivered): hollow triangle at the enqueue, solid triangle at the `remove` (the consumption point). A real answered prompt, NOT a retraction; its text rides on the solid triangle (detail pane) with an `interjection` prop. (~80% of removes, validated by reading agent text; the rest are genuine retractions, mechanically indistinguishable.) Carries no time role.
                    - **Filtered out:** an enqueue whose delivery is an `isMeta` user record (agent `/loop` / scheduled-wakeup self-prompt) ‚Üí `_synthetic`, rendered as a notification, not a prompt. Every OTHER queue op is dropped **before the y-map**, so it occupies no row and adds no gutter gap (its often ts-inverted timestamp would otherwise leave dot-less "empty gap lines").
                - ‚óÜ diamond (`.borrowed`) = untimestamped record, placed at the midpoint of its timestamped neighbors
                - ‚òÖ star (`.star`) = a collapsed context-prep burst (replay copies / compaction headers; zero work) ‚Äî **also the resume marker**: on resume the harness re-emits earlier history as replays, which collapse here (anchored at the write-moment ts). A replayed record draws at its *canonical* (earlier) position, so a parentUuid arc re-attaching across a resume legitimately spans far
                - white ring (`.sel`) = the currently selected dot (its detail is in the pane)
                - (errored records carry no ring ‚Äî an error doesn't bend the timeline; the error text still shows in the detail pane)
            - **line** (`.dl`) ‚Äî full-width dashed line; color = its time-accounting role (a first-pass stand-in for finer categories):
                - blue (`.start`) = a START ‚Äî a typed prompt
                - red (`.end`) = an END ‚Äî a `turn_duration` machine-stop (turn end / AskUserQuestion wait / notification park) or a user interrupt (also the pre-answer half of a break)
                - yellow (`.brk`) = a BREAK (a **pair**) ‚Äî (a) an AskUserQuestion / plan answer: a line on the answer dot (a START) + the preceding dot (an END); or (b) where agent activity **resumes after a stop with no interceding start** (auto-continue, a backgrounded task firing): the **first re-engagement ‚Äî thinking OR a tool call, whichever is earlier** ‚Äî gets a break (the exchange continues across it), and the preceding stop dot gets an **additional** yellow line, overriding the stop. (A tool call can resume before any thinking, so it's the honest resume point.) A dot can own multiple lines (a stop carries both its red end and the override's yellow break). Pairing is unchanged ‚Äî a break still opens/closes a block, so coverage is unaffected.
            - **line label** (`.dlbl`) ‚Äî a start line's `MM-DD ¬∑ HH:MM`, right-aligned in the gutter, centred on the line and ending well before it
            - **time-box** (`.block`) ‚Äî a full-width light-grey shade over a paired START‚ÜíEND block's y-span (hover = its duration). Pairing: a START followed by an END with no START between ‚Üí a block; stranded starts/ends are ignored but counted. A **trailing interrupted prompt** (an open START at the segment's end with no stop line ‚Äî a system restart / interruption halted record-keeping) is closed with a synthetic END at the segment's last activity record, so its work time-boxes (its internal idle is then handled by idle truncation). Each box has a **toggle** (`.boxtog` ‚ñæ/‚ñ∏, below its top timestamp): collapsing hides the box's own dots (`.nd.hid`) and **bolds its boundary lines** (`.dl.bold` ‚Äî the box's start + last line), so the un-accounted stretches stand out. Per-box state is independent (mixed states allowed); the top-bar buttons set all at once. (Relationship arcs still draw and the vertical space doesn't reflow.)
            - **active-gap fill** (`.afill`, green) ‚Äî over a short gap **between** two blocks (`< active_ceiling`), counted active (+time); never overlaps a box. **idle-gap fill** (`.ifill`, red) ‚Äî over the part of a long **in-block** gap above the `idle_ceiling`, counted idle (‚àítime); overlaps the grey box. Both follow the live active/idle ceilings (Stats tab).
            - **accounting readout** (`.acct`) ‚Äî in the segment's top headroom: `cov <covered> / span <total> ¬∑ N blocks ¬∑ M stranded ¬∑ +<active> ‚àí<idle> ‚Üí <adjusted>` (coverage + the per-segment gap adjustments)
- **detail frame** (`#detail`) ‚Äî flexes to fill from where the columns end to the viewport right edge (min 380px); `#scroll` is sized to its content so the frame **begins immediately after the columns**. Three **tabs** (`#tabs`): **Key ¬∑ Stats ¬∑ Selection**.
    - **Key tab** ‚Äî the legend (every symbol/colour) plus the **view controls**, grouped into sections:
        - **Segments** ‚Äî the kind marks (session / resume / compaction / rewind) + a description of the row fields + **collapse all / expand all** buttons (expand-all opens every segment's segment; collapse-all closes all open segments ‚Äî both via ONE batched geometry fetch + ONE relayout, not N)
        - **Time-boxes** ‚Äî the grey-block swatch + description + **collapse all / expand all** buttons (idempotent over every open segment's boxes; each box's own ‚ñæ/‚ñ∏ still toggles it individually, so a mixed state is reachable)
        - **Dots ¬∑ Lines ¬∑ Arcs ¬∑ Gutter** ‚Äî the segment-element encodings
    - **Stats tab** ‚Äî aggregate corpus totals over the from/to window (recomputed on **apply**), each section a tight grid aligned within itself; durations are `h m` (no seconds):
        - **Time** ‚Äî total (session wall-clock) ¬∑ time-boxed (Œ£ time-block coverage) ¬∑ **adjusted active** (time-boxed + active ‚àí idle)
        - **Adjustments** ‚Äî **active gaps** (+time, count) and **idle gaps** (‚àítime, count), each row inlining its own governing **ceiling** input (both knobs are ceilings ‚Äî max-gap thresholds: the active ceiling is the longest gap still filled in, the idle ceiling the longest gap still active before truncating; defaults 300s / 600s, commit on Enter/blur ‚Üí recompute + re-fetch open segments + re-color the red fills & gutter). The engaged-time smoothing, per-segment over filtered topics. Every column but the first (labels) is right-justified; columns are packed (the ceiling input sits in the next column, not floated to the edge)
        - **Counts** ‚Äî exchanges ¬∑ time-boxes
        - **Topics** ‚Äî every topic the corpus uses (`.trow` + palette swatch `.tsw`), each a toggle (off = dim + strike-through); **all** / **none** set every topic. **The totals above are topic-filtered**: the per-exchange measures (time-boxed, the adjustments, adjusted active, the counts) sum only over exchanges whose focus-thread has a *shown* topic (ANY-match), and toggling re-fetches stats + hides that topic's thread-lane stripes. So toggling everything but the billable topics yields the bill ‚Äî no billable/non-billable label anywhere. (`total` stays the unfiltered session wall-clock.)
        - footer: sessions ¬∑ segments in the window
    - **Selection tab** ‚Äî the clicked dot's (or lane bar's) detail, **auto-focused on left-click**: a metadata header (`.dmeta` ‚Äî file/ts/label/uuid/parent/‚Ä¶) and, when the record carries terminal-facing text (prompt / agent / tool i-o), a monospace subbox (`.dbody`) that **word-wraps** (vertical scroll only). Below them, the record's **membership** (`.mrow`), each its own labelled section: **Exchange** (description + record *i*/*n*), **Thread** (summary + exchange *j*/*n*), and **Topics** (the thread's tags as palette-hued chips `.tchip`). Each degrades independently ‚Äî a described-but-unthreaded exchange shows the Exchange section only. Text-selectable ‚Äî drag to select, Ctrl/‚åò+C to copy; nothing is auto-copied
- **toast** (`#toast`) ‚Äî confirms a uuid copy (right-click a node)

## Interactions

- **click** a row ‚Üí expand / collapse its event segment (first expand lazy-loads the segment; columns recompute from all open segments)
- **left-click** a dot ‚Üí switch the detail frame to the **Selection tab** and render its detail (the dot gets a white ring); the text is selectable for manual copy
- **left-click** a membership-lane bar (`.lane`) ‚Üí same Selection detail for that row's record (no dot ring), so the exchange/thread is readable straight from the lane
- **click** a tab (Key / Stats / Selection) ‚Üí switch the detail frame
- **right-click** a node ‚Üí copy its uuid (toast confirms)
- **hover** a column header ‚Üí that record class's description
- **scroll** the one container ‚Üí rows, dividers, and dots move together; the column header stays pinned (tracks horizontal scroll)
- **toggle** a topic in the Stats tab ‚Üí re-fetches the topic-filtered totals AND hides that topic's thread-lane stripes across every open segment; **all** / **none** set every topic
- **deep-link** `?seg=<start-uuid prefix>` auto-opens (and scrolls to) that segment on load; `?tab=<key|stats|sel>` selects a starting tab ‚Äî together a shareable link to a specific segment/view
