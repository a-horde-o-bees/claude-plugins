# claude-plugins — distribution mirror for the `skills` plugin

This repo publishes one flat Claude Code plugin (`skills`) to the
`a-horde-o-bees` marketplace. It is a **generated mirror**, not where skills are
edited.

## Source of truth

Skills are authored as real files in `~/.claude/skills/<name>/`, where Claude
Code reads them live — auto-discovered, refreshed mid-session, no restart. This
repo's `skills/` is a one-way mirror of that source, regenerated by
`scripts/publish-skills.py` from the allowlist in `.claude/skill-sources.json`.

- **Edit skills in `~/.claude/skills/<name>/`, never in `skills/`.** The mirror
  is build output; `publish-skills.py --check` is the drift guard.
- Work not ready to publish lives in `~/.claude/skills-under-construction/` —
  not discovered, not mirrored. Graduate a skill by moving it into
  `~/.claude/skills/` and adding its name to `.claude/skill-sources.json`.
- Flat namespace: every skill is invoked bare (`/skill-name`), no `plugin:`
  prefix, and must not shadow a built-in skill.

## Workflow

Integration is **direct** — `/git-checkpoint` commits straight to `main` and
pushes (config in `.claude/git/checkpoint.md`). Its pre-commit step runs
`publish-skills.py` so the regenerated mirror rides in the same commit; the only
required CI check is `validate` (manifest validation). There is no per-commit
version bump — the plugin's version moves only when cutting a release
(`/git-release`).

Working notes (plans, logs, task lists) and superseded material live in the
private `quarantine` repo, not here.

## Paths

| Area | Path |
|---|---|
| Skill source of truth (edit here) | `~/.claude/skills/<name>/SKILL.md` |
| Work-in-progress skills (unpublished) | `~/.claude/skills-under-construction/<name>/` |
| Published mirror (generated) | `skills/<name>/SKILL.md` |
| Plugin + marketplace manifests | `.claude-plugin/{plugin,marketplace}.json` |
| Publish allowlist | `.claude/skill-sources.json` |
| Build / validate tooling | `scripts/`, `lint/` |
